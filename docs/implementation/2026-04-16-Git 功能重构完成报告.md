## 技术架构

### 整体架构

```
SolarWireMode (三标签架构)
├── Code 标签：MonacoEditor
├── Visual 标签：SolarWirePreview
└── Version 标签：VersionView ← 新增
    └── useVersionHistory
        ├── versionCache (LRU 缓存)
        ├── workerPool (Worker 池)
        └── git-diff-analyzer.worker (后台分析)
            ↓ (getFileContent 请求)
        handleWorkerFileContentRequest
            ↓ (IPC 调用)
        git:getFileContentAtCommit
            ↓ (git show)
        git-manager.getFileContentAtCommit()
```

### 核心数据流

```
用户打开 Version 标签
    ↓
useVersionHistory 检查缓存
    ↓
命中缓存 → 直接显示结果
    ↓
未命中 → 从 Worker 池获取 Worker
    ↓
Worker 分析每个 commit
    ├─ 读取 commit 时的文件内容（通过 IPC）
    ├─ 使用 marked 解析 Markdown
    ├─ 提取 SolarWire 代码块
    ├─ 对比 AST 元素变化
    └─ 如果匹配，加入结果列表
    ↓
每 5 个提交更新一次进度
    ↓
分析完成，返回结果
    ↓
更新缓存
    ↓
释放 Worker 到池中
    ↓
更新 UI 显示结果
```

### 5. 性能优化

**LRU 缓存**：
```typescript
class VersionCache {
  private readonly MAX_SIZE = 100; // 最多 100 个文件
  private readonly TTL = 1000 * 60 * 30; // 30 分钟过期

  get(key: string): VersionCacheItem | null {
    const item = this.cache.get(key);
    if (!item) return null;
    
    // 检查是否过期
    if (Date.now() - item.timestamp > this.TTL) {
      this.cache.delete(key);
      return null;
    }
    
    return item;
  }
}
```

**Worker 池**：
```typescript
class WorkerPool {
  private readonly MAX_SIZE = 3;
  private workers: WorkerWrapper[] = [];
  private taskQueue: WorkerTask[] = [];

  acquire(): Worker {
    const idleWorker = this.workers.find(w => !w.busy);
    if (idleWorker) {
      idleWorker.busy = true;
      return idleWorker.worker;
    }

    if (this.workers.length < this.MAX_SIZE) {
      const worker = new Worker(...);
      this.workers.push({ worker, busy: true, id: this.nextId++ });
      return worker;
    }

    throw new Error('Worker pool exhausted');
  }
}
```
