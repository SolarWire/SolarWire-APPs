## 二、技术细节

### 2.1 statusStore.ts

**文件**: `src/app/stores/statusStore.ts`

```typescript
// 操作类型
type OperationType = 'save' | 'open' | 'git-commit' | 'git-push' | 'git-pull' | 'load' | 'version' | 'git-analysis';

// 操作状态
type OperationStatus = 'idle' | 'running' | 'success' | 'error';

// 状态接口
interface OperationState {
  type: OperationType;
  status: OperationStatus;
  message: string;
  errorDetail?: string; // 用于 tooltip 显示
  startTime?: number;
}
```

**核心 API**:
- `startOperation(type, message)` - 开始操作（显示进度条）
- `completeOperation(message)` - 操作成功（显示成功图标）
- `failOperation(message, errorDetail)` - 操作失败（显示错误 + tooltip）
- `setCurrentFilePath(path)` - 设置当前文件路径
- `clearOperation()` - 清除当前操作状态

### 2.2 StatusBar 组件

**文件**: `src/app/components/layout/StatusBar.tsx`

**布局结构**:
```
┌─────────────────────────────────────────────────────────────┐
| 📁 /path/to/file.md                           💾 保存成功 ✅ |
| ← 左侧：文件路径（truncate）              右侧：操作状态 →    |
└─────────────────────────────────────────────────────────────┘
```

**组件拆分**:
- `FilePathDisplay` - 显示当前文件路径（带 tooltip）
- `OperationStatusDisplay` - 显示操作状态（运行中/成功/失败）
- `Spinner` - 旋转加载图标

**状态持久化**:
- ✅ 状态不会自动清除
- ✅ 新操作覆盖旧状态
- ✅ 错误信息通过 `title` 属性实现 tooltip

### 2.4 键盘快捷键

**文件**: `src/app/components/layout/AppLayout.tsx`

```typescript
useEffect(() => {
  const handleKeyDown = (e: KeyboardEvent) => {
    const isCtrlOrCmd = e.ctrlKey || e.metaKey;
    if (isCtrlOrCmd && e.key === 's') {
      e.preventDefault();
      saveFile();
    }
  };
  window.addEventListener('keydown', handleKeyDown);
  return () => window.removeEventListener('keydown', handleKeyDown);
}, [saveFile]);
```

**支持**: `Ctrl+S` (Windows/Linux) 和 `Cmd+S` (macOS)
