# 样式问题检查报告

**日期**: 2026-04-16  
**检查范围**: SolarWire-APP/editor 前端项目  
**检查对象**: 所有 CSS 样式文件、内联样式、样式组件  

---

## 执行摘要

本次检查对项目中的所有样式相关文件进行了全面审查，发现以下主要问题：

1. **滚动条样式严重分裂** - 7 处重复定义，样式不统一
2. **内联样式使用过多** - 8 个组件使用内联样式
3. **样式变量不统一** - 部分样式未使用 CSS 变量
4. **布局问题** - 部分组件存在布局溢出风险
5. **响应式设计缺失** - 无移动端适配
6. **Tab 组件样式不统一** - 3 套不同样式，需抽象为通用组件

## 1. 滚动条样式分裂问题（严重）

### 1.1 问题描述

项目中发现 **7 套不同的滚动条样式定义**，分别位于：

| 文件路径 | 类名 | 宽度 | 状态 |
|---------|------|------|------|
| `src/app/components/ui/Scrollbar.css` | `.scrollbar` | 8px | ⚠️ 当前使用 |
| `src/app/styles/scrollbar.css` | `.scrollbar-custom` | 10px | ⚠️ 未使用 |
| `src/app/components/views/FileView.css` | `.file-view` | 10px | ⚠️ 重复 |
| `src/app/components/views/GitView.css` | `.git-view` | 10px | ⚠️ 重复 |
| `src/app/components/views/VersionView.css` | `.version-view` | 10px | ⚠️ 重复 |
| `src/app/components/views/SolarWireView.css` | `.solarwire-view` | 10px | ⚠️ 重复 |
| `src/app/components/views/RequirementView.css` | `.requirement-view` | 10px | ⚠️ 重复 |

### 1.2 样式对比

**Scrollbar.css（统一组件）**：
```css
.scrollbar::-webkit-scrollbar {
  width: 8px;  /* 8px */
  height: 8px;
}
.scrollbar::-webkit-scrollbar-track {
  background: transparent;  /* 透明背景 */
}
.scrollbar::-webkit-scrollbar-thumb {
  background: var(--border-color);  /* 使用 CSS 变量 */
  border-radius: 3px;  /* 圆角 */
}
```

**各 View 组件（重复定义）**：
```css
.xxx-view::-webkit-scrollbar {
  width: 10px;  /* 10px - 不一致 */
  height: 10px;
}
.xxx-view::-webkit-scrollbar-track {
  background: var(--bg-primary);  /* 有背景色 - 不一致 */
}
.xxx-view::-webkit-scrollbar-thumb {
  background: var(--border-color);
  border-radius: 0;  /* 无圆角 - 不一致 */
}
```

### 1.3 影响分析

- **视觉不一致**：不同视图的滚动条宽度、圆角、背景色不一致
- **维护困难**：修改滚动条样式需要修改 7 个文件
- **代码冗余**：约 140 行重复代码
- **用户体验差**：滚动条样式不统一，影响产品专业度

### 1.4 建议修复方案

**立即执行**：
1. 移除所有 View 组件中的滚动条样式定义
2. 统一使用 `Scrollbar` 组件包裹需要滚动的内容
3. 更新 `Scrollbar.css` 为统一样式

**修复后样式**：
```css
/* src/app/components/ui/Scrollbar.css */
.scrollbar::-webkit-scrollbar {
  width: 8px;
  height: 8px;
}

.scrollbar::-webkit-scrollbar-track {
  background: transparent;
}

.scrollbar::-webkit-scrollbar-thumb {
  background: var(--border-color);
  border-radius: 4px;
}

.scrollbar::-webkit-scrollbar-thumb:hover {
  background: var(--text-muted);
}
```

**预计收益**：
- 减少约 140 行重复代码
- 滚动条样式统一
- 维护成本降低 85%

## 4. 布局问题（中等）

### 4.1 发现的问题

| 问题 ID | 文件 | 问题描述 | 严重性 |
|--------|------|---------|--------|
| LAY-001 | `PropertyPanel.css` | note-section 可能溢出 | 🟡 中等 |
| LAY-002 | `SolarWireMode.css` | preview-panel 嵌套冗余 | 🟡 中等 |
| LAY-003 | `LeftPanel.css` | 条件渲染布局不稳定 | 🟡 中等 |
| LAY-004 | `MainContent.css` | 面板宽度计算不精确 | 🟢 低 |

### 4.2 详细分析

#### LAY-001: PropertyPanel note-section 溢出

**问题描述**：
note-section 的 textarea 高度自适应后，可能导致整个属性面板溢出。

**当前实现**：
```css
.note-section {
  flex-shrink: 0;
  display: flex;
  flex-direction: column;
}

.note-section textarea {
  min-height: 40px;
  max-height: none;  /* ⚠️ 无限制 */
  overflow: visible;
}
```

**建议修复**：
```css
.note-section textarea {
  min-height: 40px;
  max-height: 200px;  /* 添加最大高度限制 */
  overflow-y: auto;   /* 超过最大高度时显示滚动条 */
}
```

#### LAY-002: SolarWireMode preview-panel 嵌套冗余

**问题描述**：
已在 DOM 结构修复中移除，但 CSS 中仍存在冗余样式。

**当前实现**：
```css
/* 已不需要的样式 */
.preview-panel {
  min-width: 0;
  background: var(--bg-primary);
  /* ... */
}

.preview-content {
  height: 100%;
  /* ... */
}
```

**建议修复**：
移除上述冗余样式。

#### LAY-003: LeftPanel 条件渲染布局不稳定

**问题描述**：
当底部面板显示/隐藏时，布局可能跳动。

**当前实现**：
```tsx
{shouldShowBottomPanel && (
  <>
    <ResizableDivider />
    <div className="left-panel-bottom">
      <VersionGitTabs />
    </div>
  </>
)}
```

**建议修复**：
使用 CSS Grid 或 Flexbox 的 `flex` 属性控制高度，避免条件渲染导致的布局跳动。

**Tab.tsx 组件**：
```tsx
import React, { createContext, useContext, useState } from 'react';
import './Tab.css';

interface TabContextType {
  activeTab: string;
  setActiveTab: (id: string) => void;
}

const TabContext = createContext<TabContextType | undefined>(undefined);

interface TabListProps {
  children: React.ReactNode;
  className?: string;
}

export const TabList: React.FC<TabListProps> = ({ children, className = '' }) => {
  return (
    <div className={`tab-list ${className}`}>
      {children}
    </div>
  );
};

interface TabProps {
  id: string;
  children: React.ReactNode;
  icon?: React.ReactNode;
  badge?: number | string;
  disabled?: boolean;
  className?: string;
}

export const Tab: React.FC<TabProps> = ({
  id,
  children,
  icon,
  badge,
  disabled = false,
  className = ''
}) => {
  const context = useContext(TabContext);
  if (!context) {
    throw new Error('Tab must be used within TabProvider');
  }

  const { activeTab, setActiveTab } = context;
  const isActive = activeTab === id;

  return (
    <button
      className={`tab ${isActive ? 'active' : ''} ${className}`}
      onClick={() => setActiveTab(id)}
      disabled={disabled}
      role="tab"
      aria-selected={isActive}
    >
      {icon && <span className="tab-icon">{icon}</span>}
      <span className="tab-content">{children}</span>
      {badge && (
        <span className="tab-badge">
          {typeof badge === 'number' && badge > 99 ? '99+' : badge}
        </span>
      )}
    </button>
  );
};

interface TabPanelProps {
  id: string;
  children: React.ReactNode;
  className?: string;
}

export const TabPanel: React.FC<TabPanelProps> = ({ id, children, className = '' }) => {
  const context = useContext(TabContext);
  if (!context) {
    throw new Error('TabPanel must be used within TabProvider');
  }

  const { activeTab } = context;
  if (activeTab !== id) {
    return null;
  }

  return (
    <div className={`tab-panel ${className}`} role="tabpanel">
      {children}
    </div>
  );
};

interface TabProviderProps {
  defaultTab?: string;
  activeTab?: string;
  onTabChange?: (tabId: string) => void;
  children: React.ReactNode;
}

export const TabProvider: React.FC<TabProviderProps> = ({
  defaultTab,
  activeTab: controlledActiveTab,
  onTabChange,
  children
}) => {
  const [internalActiveTab, setInternalActiveTab] = useState(defaultTab || '');

  const activeTab = controlledActiveTab !== undefined ? controlledActiveTab : internalActiveTab;

  const setActiveTab = (tabId: string) => {
    if (controlledActiveTab === undefined) {
      setInternalActiveTab(tabId);
    }
    onTabChange?.(tabId);
  };

  return (
    <TabContext.Provider value={{ activeTab, setActiveTab }}>
      {children}
    </TabContext.Provider>
  );
};

export { TabContext };
```
