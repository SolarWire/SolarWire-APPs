# SolarWire Editor

## 目标
创建一个独立运行的 SolarWire 编辑器应用，支持代码编辑和可视化拖拽编辑两种模式，同时集成完整的 PRD 生成功能。

## 核心功能

### 1. 编辑模式
- **代码编辑模式**
  - SolarWire 语法高亮
  - 代码自动补全
  - 文件打开/保存
- **可视化拖拽编辑模式**
  - 拖拽元素到画布
  - 选中元素拖动调整位置
  - 调整元素大小
  - 元素属性面板（位置、尺寸、样式、note等）
- **双向同步**
  - 代码修改实时反映在可视化视图
  - 可视化操作实时更新代码
- **Markdown 编辑模式**
  - Markdown 语法高亮
  - 实时 Markdown 预览
  - 支持 SolarWire 代码块渲染
  - 从 Markdown 提取多个 SolarWire 代码块

### 2. 实时渲染
- 代码编辑时实时预览
- 可视化操作时实时渲染
- 支持带注释/不带注释切换
- Markdown 中的 SolarWire 代码块自动渲染

### 3. PRD 生成功能
- **多轮对话需求收集**
  - 产品类型选择（移动端/网页端/管理后台）
  - 核心功能确认
  - 详细需求确认
  - 多语言支持确认
- **生成完整 PRD 文档**
  - 产品概述
  - 功能范围
  - 业务流程图（Mermaid）
  - 交互时序图（Mermaid）
  - 页面设计
  - 页面详情（带 SolarWire 线框图）
  - 非功能性需求
- **测试用例生成**
  - 从 PRD 生成测试用例 Excel
- **多语言支持 (i18n)**
  - 按需添加多语言翻译
  - 支持多种语言（英文/中文/日文等）

### 4. 批量 SVG 生成
- 从 Markdown 提取所有 SolarWire 代码块
- 每个代码块生成两个版本：
  - `[page-name]-with-notes.svg` - 带注释
  - `[page-name]-without-notes.svg` - 不带注释
- 输出到 `.solarwire/[requirement-name]/` 目录

### 5. 四种浏览视图
- **文件视图**
  - 普通文件树结构
  - 显示所有文件和文件夹
  - 双击打开文件
  - 选择需求文档（.md）后展开其中所有 SolarWire 页面，可直接打开编辑
- **需求视图**
  - 解析整个文件夹中的所有需求文档（.solarwire/ 目录）
  - 平铺显示所有需求
  - 显示需求基本信息（项目名称、创建日期、页面数量）
  - 点击需求展开/折叠查看详情
  - 展开后显示该需求中的所有 SolarWire 页面，可直接打开编辑
- **SolarWire 视图**
  - 解析所有需求文档中的所有 SolarWire 代码块
  - 平铺显示所有 SolarWire 页面
  - 显示页面缩略图和标题
  - 点击快速跳转到对应页面进行编辑
  - 支持搜索和筛选
- **Git 视图**
  - 显示 Git 状态（已修改、暂存、未跟踪文件）
  - 显示 Git 历史提交记录
  - 显示分支列表
  - 显示远程仓库信息

### 6. UI 界面布局

**整体布局：**
1. **顶部软件导航栏**
   - 所有具体功能分组分类设计
   - 通用软件导航（文件、编辑、视图、工具、帮助等）

2. **左侧面板**
   - **上方：浏览树**
     - 可切换不同维度视图（文件视图/需求视图/SolarWire视图/Git视图）
     - 四种视图切换标签
   - **下方：版本显示**
     - **Git 视图时隐藏**
     - 文件/需求/SolarWire 视图时：
       - 未选中文件时：显示整个工作区所有版本，按时间倒序（参考 Git 版本显示）
       - 选中文件时：显示该文件/需求/页面的版本历史
       - 版本信息：版本号、时间、修改人、备注

3. **右侧编辑区（多模式）**
   - **Markdown 模式**（选中 .md 文件）
     - 左半部分：Markdown 编辑器（可编辑修改）
     - 右半部分：实时预览（支持 Markdown、Mermaid、SolarWire）
   - **SolarWire 模式**（选中页面或 .solarwire 文件）
     - 左侧：SolarWire 代码编辑器
     - 中间：SolarWire 实时预览
       - 顶部悬浮：操作工具栏
     - 右侧：元素拖拽区、选中元素属性编辑
   - **空白模式**（选中文件夹或未选中）
     - 显示空白页面和简单指引
   - **版本管理模式**（选择版本后进入）
     - 左侧：当前版本内容（只读预览）
     - 右侧：对比版本（默认上一个版本，可选择其它版本）
     - 版本差异高亮显示
   - **Git 模式**（Git 视图时）
     - Git 操作面板：
       - Git 状态（已修改、暂存、未跟踪文件）
       - 暂存/取消暂存文件
       - 提交（带提交信息）
       - 分支管理
       - 远程仓库操作（push/pull/fetch）
     - Git 历史提交记录
     - Git 分支列表
     - 文件差异对比

4. **底部状态栏**
   - 显示保存状态
   - 文件状态（已修改/已保存）
   - 当前编辑模式
   - 其他状态信息

### 7. 导出功能
- 导出单个 SVG（带/不带注释）
- 导出 PNG
- 批量导出所有 SVG
- 导出 PRD 文档（.md）
- 导出测试用例（.xlsx）

### 8. 版本管理
**设计原则：需求是主版本，页面版本从需求版本拆解**

- **需求版本（.md）**
  - 自动保存需求文档的历史版本
  - 查看需求版本历史列表
  - 恢复任意需求历史版本
  - 需求版本差异对比（diff）
  - 手动创建需求版本快照
  - 添加需求版本备注

- **页面版本（从需求版本拆解）**
  - 每个 SolarWire 页面有自己的版本历史
  - 页面版本继承自需求版本：只有当页面在该需求版本中被修改时，才会创建新版本
  - 示例：
    - 需求 v1.0 → 页面 A v1.0, 页面 B v1.0
    - 需求 v1.1（只改了页面 A）→ 页面 A v1.1, 页面 B 保持 v1.0
    - 需求 v1.2（只改了页面 B）→ 页面 A 保持 v1.1, 页面 B v1.2
    - 需求 v1.3（同时改了 A 和 B）→ 页面 A v1.3, 页面 B v1.3
  - 查看页面版本历史
  - 恢复任意页面历史版本
  - 页面版本差异对比（diff）

### 9. SolarWire 模式交互设计

**中间预览区（顶部悬浮操作工具栏）：**
- 撤销/重做
- 复制/粘贴/删除选中元素
- 对齐工具（左/中/右对齐，顶/中/底对齐）
- 带注释/不带注释预览切换

**中间预览区交互：**
- 选中元素：点击元素
- 多选：按住 Ctrl 点击 / 框选
- 拖动：选中后直接拖动
- 调整大小：选中元素后显示 8 个调整手柄（四角 + 四边中点）
- 双击元素：快速编辑文本内容（如果是带文本的元素）
- 右键菜单：复制、删除、带到前面/后面

**右侧面板：**
1. **元素库（拖拽区）**
   - 基础元素：矩形、圆角矩形、圆形、文本、线条
   - 组件：按钮、输入框、卡片、表格
   - 常用模板：登录表单、列表页、统计卡片等
   - 拖拽方式：从元素库拖到中间预览区

2. **选中元素属性面板**
   - 当选中单个元素时显示
   - 基本属性：文本内容、坐标 (x,y)、尺寸 (w,h)
   - 样式属性：bg（颜色选择器）、c（文字颜色）、b（边框）、r（圆角）、size、bold/italic、align、opacity
   - Note 编辑器：多行文本框，支持换行
   - 当选中多个元素时：只显示对齐工具和批量删除

### 10. Git 管理
- **本地 Git 管理**
  - 初始化 Git 仓库
  - 查看 Git 状态
  - 添加文件到暂存区
  - 提交变更（带提交信息）
  - 查看提交历史
  - 分支管理（创建、切换、删除分支）
  - 查看分支差异
  - 恢复到历史提交
- **远程仓库管理**
  - 克隆远程仓库
  - 添加/移除远程仓库
  - 推送到远程仓库
  - 从远程仓库拉取
  - 从远程仓库获取
  - 查看远程分支
  - 解决合并冲突

### 11. 技术选型
- 框架：Electron + React
- 代码编辑器：Monaco Editor
- 可视化画布：React + SVG 或 Canvas
- 渲染器：复用现有 SolarWire SVG 渲染器
- Markdown：marked + highlight.js
- 测试用例：xlsx 库
- Git：simple-git 或 isomorphic-git

## 目录结构规范
```
.
├── .solarwire/                    # 所有 PRD 输出根目录
│   ├── [requirement-name-1]/      # 需求1文件夹
│   │   ├── solarwire-prd.md       # PRD 文档（固定名称）
│   │   ├── [page-name]-with-notes.svg
│   │   ├── [page-name]-without-notes.svg
│   │   └── test-cases.xlsx
│   └── [requirement-name-2]/
│       └── ...
```
