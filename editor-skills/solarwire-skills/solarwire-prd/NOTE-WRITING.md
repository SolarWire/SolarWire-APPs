# SolarWire 页面笔记编写规范

## 原则

笔记是 PRD 的基础，必须：
- 每个页面一个笔记块
- 包含页面名、路径、功能、字段、交互
- 使用 SolarWire 语法描述线框

## 笔记格式

```
页面: 页面名称
路径: /page/path
功能: 简要功能描述

线框:
!title="页面名称"
!platform="mobile"
!width=375 !height=812

["元素"] @(x,y) w=w h=h

字段:
| 字段名 | 类型 | 必填 | 校验 | 说明 |

交互:
1. 操作 → 响应
```

## 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| 缺少页面路径 | 无法确定路由 | 每个页面必须有路径 |
| 线框缺少定位 | 渲染位置错误 | 每个元素必须有 @(x,y) |
| 字段描述模糊 | 开发无法实现 | 明确类型、校验、交互 |
| 页面间关系不清 | 导航结构混乱 | 明确入口和出口 |
